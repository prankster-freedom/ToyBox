# 詳細設計: モバイルズーム問題の根本修正 (Issue #2)

## 1. 目的

一部のモバイルブラウザで発生している、初期描画時の意図しないズーム（ドキュメント幅がビューポート幅を超えていると誤認される現象）を完全に解消し、ピンチ操作なしで画面にフィットさせる。

## 2. 修正方針

### 2.1. Viewport 設定の最適化 (`public/index.html`)

自動ズームを抑制し、ブラウザによる勝手なスケーリングを防止する。

- **属性**: `shrink-to-fit=no` を追加。
- **設定値**: `width=device-width, initial-scale=1.0, maximum-scale=5.0`

### 2.2. オーバーフロー制御の強化 (`public/css/style.css`)

ドキュメント幅がコンテンツ（特に背景アニメーション）によって拡張されるのを防ぐ。

- **html, body**: `overflow-x: hidden;` を強制。
- **body**: `width: 100%;` (100vw を避け、スクロールバーによる幅増加を防止)。
- **.flower-bg**:
  - `overflow: hidden;` を追加。
  - 背景の `.petal` が外側に配置されても、親要素（.flower-bg）の外にはみ出さないようにクリッピングする。

### 2.3. レスポンシブレイアウトの調整

モバイル環境（600px 以下）での要素サイズを最適化する。

- **.auth-card**:
  - `width: 90%;`
  - `max-width: 400px;`
  - `padding: 20px;` (デスクトップの 40px から縮小)
- **.auth-btn**:
  - モバイルでもタップしやすいよう、最低限の高さとフォントサイズを維持しつつ、幅が溢れないよう調整。

## 3. 影響範囲

- ログイン画面 (`auth-overlay`)
- メインチャット画面
- 日記（コレクション）画面

## 4. 検証項目

- ロード直後にズームバーが出現しないこと。
- 横スクロール（ガタつき）が発生しないこと。
- モバイル端末（縦・横）で中央のカードが画面内に収まっていること。
