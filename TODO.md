# ToyBox Project TODOs

## TODO.md 運用ルール

### 基本構造
```markdown
# Project TODOs

## 進行中 (In Progress)
- [ ] 優先度の高いタスク
  - [ ] サブタスク1
  - [ ] サブタスク2

## 待機中 (Backlog)
- [ ] 将来的に実装予定の機能A
- [ ] 技術負債の解消タスク

## 完了 (Done)
- [x] ~~完了したタスク1~~ (完了日: YYYY-MM-DD)
- [x] ~~完了したタスク2~~ (完了日: YYYY-MM-DD)
```

### 運用方針

1. **使い分けルール**
   - コード中の`// TODO:`コメント
     - 小規模な技術的TODO
     - ローカルな実装メモ
   - GitHub Issues
     - 具体的な機能実装タスク
     - バグ報告
     - 細かい改善要望
   - `TODO.md`（このファイル）
     - プロジェクト全体の方向性
     - 技術負債の一覧
     - 環境設定やドキュメント関連のTODO
     - チーム内での合意事項や検討項目

2. **更新タイミング**
   - 新しい要件やタスクが明確になった時
   - スプリントやイテレーションの開始/終了時
   - 重要な技術的決定がなされた時

3. **記載ルール**
   - 完了したタスクは削除せず、完了セクションに移動
   - 可能な限り関連するIssue番号やドキュメントへのリンクを含める
   - 詳細な実装TODOはIssuesで管理し、このファイルには参照を記載
